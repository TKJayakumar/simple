Project 1

Description of Project 1

1. **Main Menu**: Upon execution, the program presents a menu with three options: College Information, Admission, and Fees Structure.

2. **College Information**: Provides information about JK University, including its ranking, placements, and details about different departments such as Engineering, Medical, Arts and Science, and Management. Users can select a department to view more details about its admission process and cutoff marks.

3. **Admission**: Allows prospective students to apply for admission to different courses offered by JK University. Depending on the selected course (Engineering, Medical, Arts and Science, or Management), the program collects relevant information such as the student's name, email, mobile number, and subject marks. It then determines the eligibility criteria based on cutoff marks and allows the student to apply if eligible.

4. **Fees Structure**: Displays the fee structure for different courses offered by the university, including Engineering, Medical, Arts and Science, and Management. Users can select a course to view its corresponding fees.

5. **Summary**: The program seems well-structured, with clear sections for each functionality and appropriate prompts for user input. It utilizes file handling to store information about student admissions in a file named "jkinfo.doc." Additionally, it includes error handling for invalid inputs and provides options to return to the main menu or exit the program.

Project 2

Description of Project 2

The provided Java code is for an Airport Management application implemented using AWT (Abstract Window Toolkit) for GUI (Graphical User Interface). Here's a summary of its functionality:

1. **Class Structure**: The main class is named `AirportAWT`, which extends `Frame` and implements the `ActionListener` interface to handle button actions.

2. **GUI Components**: - `TextField`: Used to input the total number of runways, time remaining for planes in each runway to take off, and shelter time data. - `Button`: A submit button to initiate input of runway and shelter data, and a calculate button to compute the optimal runway for the upcoming flight. - `TextArea`: Used to display output messages and final results.

3. **Event Handling**: - The `actionPerformed` method handles button clicks. When the submit button is clicked, it reads the number of runways and creates input fields for takeoff and shelter times. When the calculate button is clicked, it computes the total time for each runway and determines the optimal runway for the next flight.

4. **Functionality**: - Upon submitting the runway and shelter data, the program calculates the total time for each runway (takeoff time + shelter time) and identifies the runway with the minimum total time, indicating the optimal runway for the next flight. - The final output includes details such as the input takeoff and shelter times, the total number of runways, and the runway number for the upcoming flight.

5. **Layout**: - Initially, the layout is set with a flow layout. - Upon submission, the layout switches to a grid layout to organize the input fields in a structured manner.

6. **Window Closing**: - The program exits gracefully when the window is closed by the user. Overall, this program provides a basic interface for managing runway and shelter data at an airport, helping to determine the optimal runway for the next flight based on the remaining time for takeoff and shelter for each runway.
